1. Optimal Hospital Location

Algorithm: 1D Median / k-Medians (approx) — Sorting + Divide & Conquer
Problem Content
Place one or multiple hospitals within a city to minimize travel distance for residents, taking into account population density and accessibility. Stakeholders include citizens, emergency services and urban planners.
Algorithmic Solution
For a single facility on a line the median minimizes the sum of absolute distances — compute by sorting positions (O(n log n)). For k facilities approximate with recursive partitioning: sort points by coordinate, split into k balanced partitions (divide-and-conquer), place facility at partition median and use a priority-queue-based local refinement (heap) for swap improvements. Weighted residents use Fenwick or prefix sums to compute marginal cost efficiently.
Input / Output (Project Version)
Input: list of resident coordinates (x,y) and optional weights. Output: coordinates of hospital(s) and estimated travel cost.
Related SDGs
SDG 3 — Good Health & Well-being SDG 11 — Sustainable Cities
C++ Example (median / simple k=1) 
// Hospital Location (1D median) - skeleton
#include <bits/stdc++.h>
using namespace std;
double median1d(vector& pts){
  sort(pts.begin(), pts.end());
  int n = pts.size();
  if(n%2) return pts[n/2];
  return (pts[n/2-1] + pts[n/2]) / 2.0;
}
int main(){
  vector pts = {1,5,3,8,9};
  cout << "Median: " << median1d(pts) << '\\n';
  return 0;
}

2. School Allocation to Students

Algorithm: Bipartite Matching / Max-Flow (Dinic) + Dijkstra for distances
Problem Content
Assign students to schools minimizing travel time while respecting school capacities and eligibility zones. Ensure fairness and capacity constraints are satisfied.
Algorithmic Solution
Model reachable pairs as a bipartite graph (students → schools). Compute distances using Dijkstra on the city graph when needed. Use Dinic's max-flow to assign students to school seats when capacities are hard constraints; for distance minimization use min-cost flow variants or repeated greedy flows. Priority queues (heaps) accelerate Dijkstra; balanced BSTs can help with tie-breaking.
Input / Output (Project Version)
Input: students (coords), schools (coords + capacity) and street graph. Output: assignment list and total travel cost.
Related SDGs
SDG 4 — Quality Education SDG 10 — Reduced Inequalities
C++ Example (Dinic skeleton) 
// Dinic skeleton for capacity assignment
#include <bits/stdc++.h>
using namespace std;
struct Edge{int to; long long cap; int rev;};
struct Dinic{
  int n; vector> g; vector level, it;
  Dinic(int n):n(n),g(n),level(n),it(n){}
  void add_edge(int u,int v,long long c){
    g[u].push_back({v,c,(int)g[v].size()});
    g[v].push_back({u,0,(int)g[u].size()-1});
  }
  bool bfs(int s,int t){ fill(level.begin(),level.end(),-1); queueq; level[s]=0; q.push(s);
    while(!q.empty()){int u=q.front(); q.pop();
      for(auto&e:g[u]) if(e.cap>0 && level[e.to]==-1){ level[e.to]=level[u]+1; q.push(e.to); }
    } return level[t]!=-1;
  }
  long long dfs(int u,int t,long long f){
    if(u==t) return f;
    for(int &i=it[u]; i<(int)g[u].size(); ++i){
      Edge &e = g[u][i];
      if(e.cap>0 && level[e.to]==level[u]+1){
        long long p = dfs(e.to,t,min(f,e.cap));
        if(p>0){ e.cap-=p; g[e.to][e.rev].cap+=p; return p; }
      }
    } return 0;
  }
  long long max_flow(int s,int t){ long long flow=0; while(bfs(s,t)){ fill(it.begin(), it.end(), 0); while(true){ long long f=dfs(s,t,LLONG_MAX); if(!f) break; flow+=f; } } return flow;}
};

3. Smart Parking Slot Allocation

Algorithm: Hungarian-like Greedy Sorting + Trie for quick permit checks
Problem Content
Assign cars to parking slots to minimize total walking distance while satisfying capacity and accessibility rules — e.g., handicap slots, reserved permits.
Algorithmic Solution
Create list of (car,slot,distance), sort ascending by distance (O(m log m)). Greedily assign smallest pairs while marking cars/slots used. Use Trie structure for permit/license prefix checks to efficiently filter eligible cars for reserved slots. Balanced trees (AVL) can manage dynamic slot availability.
Input / Output (Project Version)
Input: car list, slot list, constraints. Output: assignment and total walking distance.
Related SDGs
SDG 11 — Sustainable Cities SDG 9 — Industry & Infrastructure
C++ Example (greedy sorted pairs) 
// Greedy assign from sorted pairs (skeleton)
#include <bits/stdc++.h>
using namespace std;
struct Pair{int car, slot; double dist;};
int main(){
  vector pairs = {{0,0,2.3},{1,2,1.1},{2,1,3.0}};
  sort(pairs.begin(), pairs.end(), [](auto &a, auto &b){ return a.dist < b.dist; });
  vector carUsed(100,false), slotUsed(100,false);
  for(auto &p: pairs){
    if(!carUsed[p.car] && !slotUsed[p.slot]){
      carUsed[p.car]=slotUsed[p.slot]=true;
      cout << "Assign car " << p.car << " -> slot " << p.slot << "\\n";
    }
  }
  return 0;
}

4. Industrial vs Residential Separation

Algorithm: Graph Coloring (BFS/DFS) + Union-Find for components
Problem Content
Assign zoning types across parcels so incompatible land uses don't share edges, preserving safety and regulation.
Algorithmic Solution
Model parcels as nodes and adjacencies as edges. Use BFS/DFS to attempt 2-coloring (bipartite check). If conflicts remain, group parcels via union-find and apply local backtracking on small conflict components to resolve constraints using taught primitives only.
Input / Output (Project Version)
Input: parcel adjacency graph and attributes. Output: zone color per parcel, and conflict report if any.
Related SDGs
SDG 11 — Sustainable Cities SDG 3 — Good Health

6. Land Subdivision Optimization

Problem Content

Algorithmic Solution

7. High Footfall Commercial Zone Prediction

Algorithm: Grid Binning + 2D Prefix / Fenwick + Sorting
Problem Content
Predict commercial hotspots by analyzing movement traces, POI density and temporal patterns — useful for planning retail and transit capacity.
Algorithmic Solution
Quantize region into grid cells and aggregate traces into counts. Build 2D prefix sums (or 2D Fenwick) to query sub-rectangles efficiently. Rank candidate regions by aggregated counts (sort) to identify hotspots. Temporal sliding windows use time-indexed Fenwick updates for efficient changes.
Input / Output (Project Version)
Input: movement traces (x,y,t), POI data. Output: ranked hotspot cells and suggestions for interventions.
Related SDGs
SDG 8 — Decent Work & Economic Growth SDG 11 — Sustainable Cities
C++ Example (2D prefix sums) 
// 2D prefix sums skeleton
#include <bits/stdc++.h>
using namespace std;
vector> build_prefix(const vector>& a){
  int n=a.size(), m=a[0].size();
  vector> ps(n+1, vector(m+1, 0));
  for(int i=1;i<=n;i++) for(int j=1;j<=m;j++) ps[i][j] = a[i-1][j-1] + ps[i-1][j] + ps[i][j-1] - ps[i-1][j-1];
  return ps;
}
long long rect_sum(const vector>& ps,int r1,int c1,int r2,int c2){
  return ps[r2+1][c2+1] - ps[r1][c2+1] - ps[r2+1][c1] + ps[r1][c1];
}

9. Footpath Connectivity Plan

Algorithm: Shortest Paths (Dijkstra) + Bridge/Articulation detection (DFS)
Problem Content
Detect disconnected pedestrian networks, compute shortest safe routes and recommend minimal new links to improve connectivity.
Algorithmic Solution
Construct pedestrian graph; run Dijkstra for shortest routes. Use DFS to find bridges and articulation points that indicate fragile connectivity. For minimal link suggestions compress components and run Kruskal-like MST on compressed graph to choose cheapest connecting edges using union-find — all taught algorithms only.
Input / Output (Project Version)
Input: street graph, pedestrian counts, safety constraints. Output: shortest-path matrices, suggested new footpath links and estimated improvement metrics.
Related SDGs
SDG 11 — Sustainable Cities SDG 3 — Good Health
C++ Example (bridge detection) 
// Bridge detection via DFS (skeleton)
#include <bits/stdc++.h>
using namespace std;
void dfs(int u,int p,const vector>& g,vector& tin,vector& low,vector& vis,int &timer,vector>& bridges){
  vis[u]=1; tin[u]=low[u]=++timer;
  for(int v: g[u]){
    if(v==p) continue;
    if(vis[v]) low[u]=min(low[u], tin[v]);
    else{
      dfs(v,u,g,tin,low,vis,timer,bridges);
      low[u]=min(low[u], low[v]);
      if(low[v] > tin[u]) bridges.push_back({u,v});
    }
  }
}

10. Underground Utility Routing

Algorithm: A* (heuristic) or Dijkstra + Greedy disjoint routing
Problem Content
Plan routing of underground utilities (water, sewer, fiber, power) to avoid spatial conflicts and critical infrastructure while minimizing construction cost and length.
Algorithmic Solution
Model grid with obstacles. For each route run A* (with Manhattan/Euclidean heuristic) or Dijkstra to find shortest path. After fixing one utility, mark its corridor cells and route next utility greedily. For multiple-commodity constraints run iterative rerouting using priority queues. All done with taught pathfinding primitives only.
Input / Output (Project Version)
Input: grid map with obstacles and existing utilities, start & end coordinates. Output: set of routes avoiding prohibited cells with cost metrics.
Related SDGs
SDG 9 — Industry & Infrastructure SDG 6 — Clean Water & Sanitation
C++ Example (A* skeleton) 
// A* skeleton (Manhattan heuristic)
#include <bits/stdc++.h>
using namespace std;
int heuristic(pair a, pair b){ return abs(a.first-b.first) + abs(a.second-b.second); }
vector> astar(const vector>& grid, pair start, pair goal){
  int n=grid.size(), m=grid[0].size();
  using Node = tuple; // f,g,r,c
  priority_queue, greater> open;
  map, pair> came;
  map, int> gscore;
  open.push({heuristic(start,goal), 0, start.first, start.second});
  gscore[start]=0;
  int dr[4]={1,-1,0,0}, dc[4]={0,0,1,-1};
  while(!open.empty()){
    auto [f,g,r,c]=open.top(); open.pop();
    pair cur={r,c};
    if(cur==goal){
      vector> path; auto curp=cur;
      while(came.count(curp)){ path.push_back(curp); curp=came[curp]; }
      path.push_back(start); reverse(path.begin(), path.end()); return path;
    }
    for(int k=0;k<4;k++){
      int nr=r+dr[k], nc=c+dc[k];
      if(nr>=0 && nr=0 && nc nxt={nr,nc}; int tentative = g + 1;
        if(!gscore.count(nxt) || tentative < gscore[nxt]){
          gscore[nxt]=tentative; came[nxt]=cur;
          open.push({tentative + heuristic(nxt,goal), tentative, nr, nc});
        }
      }
    }
  }
  return {};
}
