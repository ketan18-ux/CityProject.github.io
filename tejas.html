Land Use, Zoning & Location Optimization Projects

These 10 problems are expressed using only the algorithms and data structures taught in class — recursion, sorting, trees, heaps, segment/fenwick trees, graph algorithms, string search, and related primitives.

1. Optimal Hospital Location

Algorithm: Median, Sorting & Divide-and-Conquer
Problem Content
Place one or multiple hospitals within a city to minimize travel distance for residents, taking into account population density and accessibility.
Algorithmic Solution
For one facility on a line, the median minimizes the sum of absolute distances — compute by sorting positions. For multiple facilities, use recursive partitioning (divide-and-conquer): sort points by x (or principal axis), split into balanced groups, place central facility at group's median. Refine with local improvements using a priority queue (heap) for candidate swaps. Weighted populations use prefix sums or Fenwick trees to compute marginal gains.

2. School Allocation to Students

Algorithm: Shortest Paths + Greedy Assignment (Dijkstra + Heap)
Problem Content
Assign students to schools minimizing travel time while respecting school capacities and eligibility zones.
Algorithmic Solution
Model the area as a graph, run Dijkstra from each school (or multi-source) to compute distances. Use a min-heap of (distance, student, school) pairs and greedily assign students to nearest schools while decrementing capacities. Use balanced trees (AVL/RB) to track tie-breaks and to support quick updates for prioritized fairness criteria.

3. Smart Parking Slot Allocation

Algorithm: Greedy Assignment + Sorting (with Trie for lookups)
Problem Content
Assign incoming cars to parking slots to minimize total walking distance while satisfying accessibility and permit constraints.
Algorithmic Solution
Compute distances for candidate car-slot pairs and sort them. Greedily select smallest-distance pairs, reserving cars and slots (similar to assignment via greedy). Use a Trie for fast prefix-based lookups (permit/id matching). Maintain available slots using a BST (AVL/RB) if dynamic removal is needed.

4. Industrial vs Residential Separation

Algorithm: Graph Coloring via BFS/DFS + Union-Find
Problem Content
Assign zoning types across adjacent parcels so incompatible types do not share edges, ensuring safety and regulatory constraints.
Algorithmic Solution
Model parcels as graph nodes and adjacency as edges. Use BFS/DFS to test 2-colorability (bipartite). For conflicts requiring group decisions, use union-find to manage connected components and backtracking to resolve small conflicting sets. This uses only standard classroom graph primitives.

5. Public Park Placement Optimization

Algorithm: Greedy Max-Coverage + Fenwick / Segment Tree
Problem Content
Choose k park sites from candidate locations to maximize the population covered within a walking radius.
Algorithmic Solution
Discretize city into grid cells. Precompute coverage sets per candidate site. Iteratively pick site with greatest marginal coverage (greedy). Use Fenwick or segment tree to maintain and query remaining uncovered population counts efficiently for marginal gain calculations.

6. Land Subdivision Optimization

Algorithm: Recursion, Sorting & Geometry (Shoelace area)
Problem Content
Partition large land parcels into smaller, balanced plots while preserving constraints like road access and minimum area.
Algorithmic Solution
Use recursive divide-and-conquer guided by sorted vertex projections. Compute polygon areas via shoelace formula to balance splits. Recurse until subplots meet constraints. Use BFS on adjacency of final subplots to ensure connectivity.

7. High Footfall Commercial Zone Prediction

Algorithm: Grid Binning + 2D Prefix / Fenwick + Sorting
Problem Content
Predict commercial hotspots by analyzing movement traces, POI density, and temporal patterns.
Algorithmic Solution
Bin traces into a regular grid. Build 2D prefix sums (or 2D Fenwick) to query sub-rectangles quickly and compute aggregated footfall for candidate zones. Sort candidate regions by footfall to rank hotspots. Time windows use sliding window sums or time-indexed Fenwick updates.

8. Creation of Green Belt Buffers

Algorithm: Multi-source BFS Layers + Masking
Problem Content
Create buffer belts around polluted zones or city limits to protect ecosystems and form green corridors.
Algorithmic Solution
Represent map as grid, run multi-source BFS from pollution cells to compute layer distances (buffer bands). Respect obstacles and protected polygons by masking them during BFS expansion. For weighted cost bands, substitute Dijkstra (priority queue).

9. Footpath Connectivity Plan

Algorithm: Dijkstra + Bridge/Articulation Detection (DFS)
Problem Content
Detect disconnected pedestrian networks, compute safe shortest routes, and recommend minimal new links to improve connectivity.
Algorithmic Solution
Use Dijkstra for shortest paths and DFS-based bridge/articulation detection to find vulnerabilities. To suggest minimal new links, compress components and run Kruskal-like MST (union-find) on possible connecting edges to pick cheapest additions that join components.

10. Underground Utility Routing

Algorithm: A* / Dijkstra + Greedy Disjoint Routing
Problem Content
Plan underground utility routes to avoid spatial conflicts with minimal total length and cost.
Algorithmic Solution
Model grid graph and run A* (heuristic) or Dijkstra for each route. After routing one utility, mark corridor cells as blocked and route successive utilities (greedy sequencing). For better disjointness, allow limited rerouting iterations using priority queues. All steps use standard taught algorithms.
