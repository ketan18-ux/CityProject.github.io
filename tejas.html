1. Optimal Hospital Location

Algorithm: 1D Median / k-Medians (approx)
Problem Content
Place one or multiple hospitals within a city to minimize travel distance for residents, taking into account population density and accessibility. Stakeholders include citizens, emergency services and urban planners.
Algorithmic Solution
For the single-hospital (1D) case, the median minimizes absolute distance. For k facilities, run k-medians or k-means-like iterative improvement with assignment and center update steps; use heuristics for initialization.
Input / Output (Project Version)
Input: List of resident coordinates (x,y) and optional weights (population). Output: Coordinates of hospital locations and total travel cost estimate.
Related SDGs
SDG 3 — Good Health & Well-being SDG 11 — Sustainable Cities
C++ Example (median / simple k=1) 
// Hospital Location (1D median)
#include <bits/stdc++.h>
using namespace std;

double median_1d(vector& pts){
    sort(pts.begin(), pts.end());
    int n = pts.size();
    if(n % 2 == 1) return pts[n/2];
    return (pts[n/2 - 1] + pts[n/2]) / 2.0;
}

int main(){
    vector pts = {1,5,9,3,8};
    cout << fixed << "Median position: " << median_1d(pts) << '\\n';
    return 0;
}

2. School Allocation to Students

Algorithm: Bipartite Matching / Max-Flow (Dinic)
Problem Content
Assign students to schools minimizing travel time while respecting school capacities and eligibility zones. Ensure fairness and capacity constraints are satisfied.
Algorithmic Solution
Model as bipartite graph (students → schools) with edges for reachable schools. Use max flow (Dinic) with capacities to enforce school seat limits and minimize cost via repeated flows or min-cost max-flow when costs vary.
Input / Output (Project Version)
Input: Students with coordinates, schools with coordinates and capacities; optionally cost matrix (distance). Output: Assignment list (student -> school) and total travel cost.
Related SDGs
SDG 4 — Quality Education SDG 10 — Reduced Inequalities
C++ Example (Dinic skeleton) 
// Dinic's Algorithm (C++) - skeleton for student-school assignment
#include <bits/stdc++.h>
using namespace std;
struct Edge{int to; long long cap; int rev;};
struct Dinic {
    int n; vector> g; vector level, it;
    Dinic(int n):n(n),g(n),level(n),it(n){}
    void add_edge(int u,int v,long long c){
        g[u].push_back({v,c,(int)g[v].size()});
        g[v].push_back({u,0,(int)g[u].size()-1});
    }
    bool bfs(int s,int t){ fill(level.begin(), level.end(), -1); queueq; level[s]=0; q.push(s);
        while(!q.empty()){ int u=q.front(); q.pop(); for(auto &e:g[u]) if(e.cap>0 && level[e.to]==-1){ level[e.to]=level[u]+1; q.push(e.to); } }
        return level[t]!=-1;
    }
    long long dfs(int u,int t,long long f){ if(u==t) return f;
        for(int &i=it[u]; i<(int)g[u].size(); ++i){ Edge &e = g[u][i];
            if(e.cap>0 && level[e.to]==level[u]+1){ long long ret = dfs(e.to,t,min(f,e.cap)); if(ret>0){ e.cap-=ret; g[e.to][e.rev].cap+=ret; return ret; } }
        } return 0;
    }
    long long max_flow(int s,int t){ long long flow=0;
        while(bfs(s,t)){ fill(it.begin(), it.end(), 0); while(true){ long long f=dfs(s,t,LLONG_MAX); if(f==0) break; flow+=f; } } return flow;
    }
};

3. Smart Parking Slot Allocation

Algorithmic Solution

4. Industrial vs Residential Separation

Algorithm: Graph Coloring / Bipartite Check
Problem Content
Assign zoning types (residential/industrial) across adjacent land parcels so incompatible types do not touch edge-to-edge. This prevents harmful co-location of industries and homes.
Algorithmic Solution
Model parcels as nodes and adjacency as edges. A bipartite graph implies a 2-coloring exists. For general planar zoning, graph-coloring heuristics or constraint solvers (SAT/ILP) are used.
Input / Output (Project Version)
Input: Parcel adjacency graph and parcel attributes. Output: Coloring assignment (zone per parcel) and a feasibility report highlighting conflicts if any.
Related SDGs
SDG 11 — Sustainable Cities SDG 3 — Good Health

9. Footpath Connectivity Plan

Algorithm: Shortest Paths / Graph Connectivity
Problem Content
Detect disconnected pedestrian networks, compute shortest safe routes between origins and destinations, and recommend minimal new links to improve connectivity.
Algorithmic Solution
Use graph constructions with nodes at intersections and weights from pedestrian safety metrics. Dijkstra for shortest paths, bridge/ articulation point detection for vulnerabilities, and minimum link addition via connectivity augmentation heuristics.
Input / Output (Project Version)
Input: Street graph, pedestrian counts, safe path constraints. Output: Shortest-path matrices, suggested new footpath links and estimated improvement metrics.
Related SDGs
SDG 11 — Sustainable Cities SDG 3 — Good Health
C++ Example (Dijkstra) 
// Dijkstra shortest paths (adjacency map)
#include <bits/stdc++.h>
using namespace std;
unordered_map dijkstra(const unordered_map>>& graph, int start){
    unordered_map dist;
    for(auto &kv: graph) dist[kv.first] = LLONG_MAX;
    dist[start]=0;
    priority_queue, vector>, greater>> pq;
    pq.push({0,start});
    while(!pq.empty()){
        auto [d,u] = pq.top(); pq.pop();
        if(d>dist[u]) continue;
        auto it = graph.find(u);
        if(it == graph.end()) continue;
        for(auto [v,w]: it->second){
            long long nd = d + w;
            if(nd < dist[v]){ dist[v]=nd; pq.push({nd,v}); }
        }
    }
    return dist;
}

10. Underground Utility Routing

Algorithm: A* Search / Disjoint Shortest Paths
Problem Content
Plan the routing of underground utilities (water, sewer, fiber, power) to avoid spatial conflicts and critical infrastructure while minimizing construction cost and length.
Algorithmic Solution
Model as grid/graph with obstacles. Use A* for pathfinding with Manhattan or Euclidean heuristics. For multiple utilities with disjointness constraints, use successive routing with reserved corridors or multi-commodity flow approximations.
Input / Output (Project Version)
Input: Grid map with obstacles and existing utilities, start & end coordinates for each route. Output: Set of routes avoiding prohibited cells, length and cost metrics, and clash-free corridor map.
Related SDGs
SDG 9 — Industry & Infrastructure SDG 6 — Clean Water & Sanitation
C++ Example (A* on grid) 
// A* Search on grid (Manhattan heuristic)
#include <bits/stdc++.h>
using namespace std;
int heuristic(pair a, pair b){ return abs(a.first-b.first) + abs(a.second-b.second); }
vector> astar(const vector>& grid, pair start, pair goal){
    int n = grid.size(), m = grid[0].size();
    using Node = tuple; // f, g, r, c
    priority_queue, greater> open;
    map, pair> came_from;
    map, int> gscore;
    open.push({heuristic(start,goal), 0, start.first, start.second});
    gscore[start]=0;
    const int dr[4]={1,-1,0,0}; const int dc[4]={0,0,1,-1};
    while(!open.empty()){
        auto [f,g,r,c] = open.top(); open.pop();
        pair cur={r,c};
        if(cur==goal){
            vector> path; auto curp=cur;
            while(came_from.count(curp)){ path.push_back(curp); curp = came_from[curp]; }
            path.push_back(start);
            reverse(path.begin(), path.end()); return path;
        }
        for(int k=0;k<4;++k){ int nr=r+dr[k], nc=c+dc[k];
            if(nr>=0 && nr=0 && nc nxt={nr,nc}; int tentative = g + 1;
                if(!gscore.count(nxt) || tentative < gscore[nxt]){
                    gscore[nxt]=tentative; came_from[nxt]=cur;
                    open.push({tentative + heuristic(nxt,goal), tentative, nr, nc});
                }
            }
        }
    }
    return {};
}
