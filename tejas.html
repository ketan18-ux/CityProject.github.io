Land Use, Zoning & Location Optimization (with Algorithms Taught in Class)
1. Optimal Hospital Location
Concept Used: Math Foundations + Greedy Strategy + Recursion (Median Finding)

To minimize average travel distance, the problem reduces to finding a 1-D or 2-D median location.

If sorted list is used → Quick Sort / Merge Sort to sort coordinates. Median is computed efficiently using a divide-and-conquer selection algorithm.
2. School Allocation to Students
Concept Used: Graph Matching + Recursion Overhead + Greedy

Students and schools form a bipartite graph. We compute minimum travel assignment using:
• Brute-Force String Search–like matching iteration • Greedy improvement until stable
3. Smart Parking Slot Allocation
Concept Used: Dynamic Programming + Sorting + Greedy

Cars and parking slots are sorted using Heap Sort / Merge Sort. Minimum walking distance is achieved using a DP cost-minimization table (Lookup Table).
4. Industrial vs Residential Separation
Concept Used: Graph Coloring + Recursion + BFS/DFS

Regions are nodes, adjacency forms edges. Use 2-coloring (like bipartite test) using
• DFS (recursive), or • BFS Queue
5. Public Park Placement Optimization
Concept Used: Max Coverage + Greedy + Segment Tree / Fenwick Tree

People distribution over regions is stored in a Fenwick Tree for fast range queries. A greedy max-coverage algorithm selects k-locations maximizing total served population.
6. Land Subdivision Optimization
Concept Used: Divide & Conquer + Recursion + Binary Tree Splitting

Large land is recursively split into two valid subplots (like Quick Sort partitioning). A Binary Tree represents subdivision hierarchy.
7. High Footfall Commercial Zone Prediction
Concept Used: Clustering + Heaps + Sorting

Data points representing visitor density are grouped via:
• Heap-based cluster centroid updates
• Sorting (Merge/Quick Sort) for region grouping
8. Creation of Green Belt Buffers
Concept Used: Constraint Solving + Backtracking + Recursion

Buffer zones around industries are placed using constraints (minimum required distance). Backtracking tries all placements, pruning invalid ones early.
9. Footpath Connectivity Plan
Concept Used: Graph Traversal + BFS/DFS + Floyd/Warshall

To identify disconnected pedestrian pathways:
• DFS/BFS finds connected components • Floyd–Warshall gives all-pairs shortest paths • Dijkstra computes shortest walking routes
10. Underground Utility Routing
Concept Used: Shortest Path Algorithms (Dijkstra, Bellman-Ford) + Tree Structures

Pipes/cables must not intersect. The routing uses:
• Dijkstra for cost-optimal paths • Bellman–Ford if negative costs exist • Trie-like path compression for storing reusable segments
