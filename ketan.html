Problem

The city wants to connect all important junctions with new roads. Budget is limited, so we must avoid redundant roads while still guaranteeing that any pair of junctions is reachable.

Practical Solution

Model junctions as vertices and candidate roads as weighted edges (cost or distance). Using an MST algorithm, we select a subset of roads that connects all vertices with minimum total cost and no cycles. This directly becomes the proposed road construction plan.

Input (Example)

Number of junctions n, number of candidate roads m, followed by m lines: u v cost.

Output

List of roads chosen in MST and the minimum total construction cost.

Kruskal's MST Prim's MST Union‚ÄìFind

With sorting and Union‚ÄìFind, Kruskal runs in O(m log n). Space complexity is O(n + m).

Problem

Citizens struggle to pick the fastest route because some short roads are heavily congested. We must account for time, not just distance.

Practical Solution

Represent each junction as a node and each road segment as a directed edge with weight equal to estimated travel time (from live / historical data). Running Dijkstra from the source zone gives minimum-time paths to all other zones.

Input (Example)

Graph with n nodes, m directed edges, weight = travel time, plus source node.

Output

Shortest travel time and route (sequence of nodes) from source to every other node.

Algorithms & Techniques
Dijkstra using Min-Heap Greedy Relaxation
Complexity

With adjacency list & binary heap, complexity is O((n + m) log n). Space: O(n + m).

Impact: Reduces average commute time during peak hours for regular travellers.
Problem

Fixed green times cause long queues on some arms while others remain lightly loaded. Signals must adapt to varying traffic flow.

Practical Solution

Each lane is treated as a job requiring green time proportional to its queue length. Using greedy ideas similar to interval scheduling, the green slices are allocated dynamically, giving priority to lanes with highest waiting time while preventing starvation.

Input (Example)

Queue lengths and arrival rates for each lane over a time window.

Output

Sequence of green phases and durations for each lane within one full signal cycle.

Algorithms & Techniques
Greedy Scheduling Priority Queue by Queue Length
Complexity

Each update of the schedule is roughly O(k log k) where k is number of lanes at the junction.

Impact: Decreases average waiting time and smooths traffic flow at busy intersections.
Problem

During emergencies, usual routes might be blocked due to jams or accidents. Vehicles must get a route that is both fast and currently feasible.

Practical Solution

Maintain a graph with live edge weights and blocked edges. Use A* search from the emergency station to the incident, combining travelled time so far with a heuristic (straight-line distance) to explore best promising paths first.

Input (Example)

Graph, source & target nodes, travel times, and a list of temporarily blocked edges.

Output

Feasible route with minimal estimated travel time plus backup alternative route.

Algorithms & Techniques
A* Search Heuristic Design Real-time Graph Updates
Complexity

In the worst case similar to Dijkstra: O((n + m) log n), but usually faster due to heuristic pruning.

Impact: Reduces response time to critical incidents and saves lives.
Problem

Some neighborhoods receive low water pressure during peak hours. The department wants to know which pipes limit overall supply.

Practical Solution

Model the network as a flow graph with a source (treatment plant) and sinks (zones), each pipe edge having a capacity. Running a max-flow algorithm reveals the total possible supply and the min-cut edges that saturate, which are precisely the bottleneck pipes.

Input (Example)

Flow network: nodes, directed pipes with capacities, source and sink nodes.

Output

Maximum flow value and set of saturated edges forming the min-cut.

Algorithms & Techniques

Problem

Extending the power grid to new areas is costly. The utility wants a configuration that minimizes the length and loss of transmission lines.

Practical Solution

Substations and new zones are nodes; candidate high-tension lines are weighted edges using a combination of distance and expected power loss. An MST over this graph suggests which lines to build to supply all zones at minimum cost.

Input (Example)

Nodes with coordinates, candidate connections with cost = f(distance, loss).

Output

Set of chosen transmission lines and overall cost / loss estimate.

Algorithms & Techniques

Problem

Manually planned routes cause unnecessary travelling and some bins remain full for long.

Practical Solution

Model pickup locations as cities in a Travelling Salesman Problem, with the depot as start/end. A greedy nearest-neighbour heuristic followed by 2-opt improvement creates a short route that visits all pickup points once and returns to the depot.

Input (Example)

Coordinates or distance matrix for depot and all pickup locations.

Output

Order of visiting locations (cycle) and total distance of the route.

Algorithms & Techniques
Greedy TSP 2-opt Local Search
Complexity

Greedy construction: O(n¬≤). 2-opt improvement: worst case O(n¬≤) per pass.

Problem

There are many candidate trips but limited buses. Overlapping trips create shortages on some routes while buses remain idle elsewhere.

Practical Solution

Each possible trip is an interval with start and end time. Using greedy interval scheduling (sorting by finish time), we assign each bus a maximum set of non-overlapping trips, which forms the timetable.

Input (Example)

List of trips: route id, start time, end time, required bus count.

Output

Assignment of trips to buses and final timetable matrix.

Algorithms & Techniques
Interval Scheduling Greedy Sorting by Finish Time
Complexity

Sorting trips: O(k log k), where k is number of trips.

Placeholder for timetable grid view for buses across the day.
Impact: Maximizes useful bus trips without increasing fleet size.
Problem

Some bridges are much more critical than others. If they fail, entire regions become isolated. The city wants a ranked list of such bridges.

Practical Solution

Model road network as an undirected graph; edges corresponding to bridges are analysed using bridge/ articulation-edge algorithms or min-cut computations. Edges whose removal increases the number of connected components are marked as critical.

Input (Example)

Graph of regions and roads; subset of edges labelled as bridges.

Output

List of critical bridges along with importance score / affected population.

Algorithms & Techniques
Tarjan Bridge Algorithm Min-Cut Interpretation
Complexity

Bridge detection using DFS runs in O(n + m).

Placeholder for river bridges graph where critical bridges are highlighted.
Impact: Helps plan reinforcement budget for the most risky bridges first.
Problem

During heavy rain, low-lying localities flood first. The city needs to know how much time is available to move residents to safe shelters.

Practical Solution

Localities are nodes and roads are edges. Multi-source BFS is run from all initial flooded locations. The BFS level of a node equals the time step when water reaches that locality, which is used to mark safe vs unsafe shelters and evacuation routes.

Input (Example)

Graph of localities, list of flood-source nodes, and list of shelters.

Output

Time for water to reach each locality and recommended safe shelter per locality.

Algorithms & Techniques
BFS Multi-source BFS
Complexity

BFS runs in O(n + m) time and O(n) space.

Impact: Supports evacuation planning and disaster-risk reduction for vulnerable wards.
