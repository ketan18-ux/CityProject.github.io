3. Traffic Signal Timing Optimization
Junction Management Queues Â· Priority Queue Â· Efficient Sorting

Problem Context

Fixed green-light durations at urban junctions often fail to match real traffic demand. Some approaches remain congested while others receive unused green time, leading to increased waiting time, fuel consumption, and emissions.

The objective is to dynamically allocate green time based on observed queue lengths at each approach, ensuring fair and efficient traffic flow.

Algorithmic Solution

Each lane is modeled as a queue representing waiting vehicles. At the start of each signal cycle:

  • Queue lengths are measured and stored in an array.
  • Lanes are ranked using efficient sorting (Merge Sort / Quick Sort).
  • A max-priority queue is used to allocate additional green time to the most congested lanes.

This greedy yet efficient strategy adapts in real time while remaining fast enough for live signal control.

Input / Output

Input

Queue length of each lane (North, South, East, West) and total signal cycle time.

Output

Computed green time allocation for each lane for the current cycle.

4. Emergency Vehicle Fastest Route Planner
Emergency Response BFS + DFS

Problem Context

During accidents, fires, or medical emergencies, certain roads may become blocked due to congestion or damage. Emergency vehicles must identify whether a feasible route exists and, if so, reach the destination as quickly as possible.

Algorithmic Solution

The road network is treated as an unweighted graph after removing blocked roads.

  • DFS: Used to verify connectivity between the emergency station and the incident location.
  • BFS: Finds the minimum-hop route, which approximates the fastest path when each road segment has similar traversal time.

The combined approach is efficient, robust, and suitable for real-time emergency planning.

Input / Output

Input: Graph of roads, list of blocked roads, station node, incident node.

Output: Reachability status, shortest hop count, and one feasible path.

9. Bridge Construction Priority Model
Critical Infrastructure DFS Bridge Detection (Tarjan Method)

Problem Context

Some bridges act as single points of failure in the transport network. If such a bridge collapses or is closed for repair, large regions may become disconnected from hospitals, schools, or workplaces.

The goal is to identify critical bridges whose failure would disconnect the road network, so they can be prioritised for inspection and reinforcement.

Algorithmic Solution

The road network is modeled as an undirected graph. We use a DFS-based bridge detection algorithm:

  • Each node is assigned a discovery time disc[u] and a lowest reachable time low[u].
  • An edge (u, v) is a bridge if low[v] > disc[u].
  • The algorithm runs in O(V + E) time.

Input / Output

Input: Undirected graph of regions connected by roads/bridges.

Output: List of bridges that are critical links.

10. Flood Evacuation Simulation
Disaster Management Multi-source BFS

Problem Context

Flooding spreads from rivers and low-lying areas to surrounding localities. Authorities must identify which shelters remain reachable before floodwaters cut off access roads.

Algorithmic Solution

Flood sources are treated as multiple BFS starting points:

  • All flood source nodes are enqueued with time 0.
  • Each BFS level represents flood spread per time unit.
  • Shelters are evaluated based on flood arrival time.

Time Complexity: O(V + E).

Input / Output

Input: City graph, flood sources, shelter nodes, time threshold.

Output: Flood arrival time and shelter safety classification.
