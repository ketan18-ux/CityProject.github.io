Optimal Road Network Construction
Zone: City-wide ¬∑ Category: Transport Infrastructure
View code (update link)
Conceptual MST of city junctions. Each node is a major junction; edges are candidate roads with construction cost.
MST Layout

1. Problem

The city wants to connect all major junctions with new roads. Budget is limited, so we must connect every location while keeping total construction cost minimal and avoiding unnecessary duplicate roads.

2. Practical Solution

We model junctions as vertices and possible roads as weighted edges (cost or distance). Using a Minimum Spanning Tree algorithm, we select a subset of roads that connects all junctions with minimum total cost, guaranteeing there is exactly one simple path between any pair.

3. Algorithm(s) Used

Kruskal's MST Prim's MST Union-Find DS
Shortest Travel-Time Routing
Zone: Commuter ¬∑ Category: Route Planning
View code (update link)
Weighted directed graph where edge weights store travel time based on traffic intensity.
Dijkstra Graph

1. Problem

Citizens struggle to find the fastest route between two zones during peak hours. Distance alone is not enough because some shorter routes are heavily congested.

2. Practical Solution

Each road segment is a directed edge with weight equal to estimated travel time. Running Dijkstra's algorithm from the source zone, we compute minimum-time paths to all other zones. The system returns the route with least travel time and can be updated as traffic data changes.

3. Algorithm(s) Used

Dijkstra Binary Heap / Priority Queue
Traffic Signal Timing Optimization
Zone: Junctions ¬∑ Category: Flow Management
View code (update link)
Intersection model with incoming lanes treated as jobs competing for green signal slots.
Scheduling View

1. Problem

Major junctions get long queues on some arms while other arms remain under-utilized. Fixed green signal times are not optimal for dynamic traffic.

2. Practical Solution

Treat each direction as a job with processing time proportional to queue length. Using greedy or interval scheduling ideas, we allocate green time slices to lanes to minimize total waiting time and prevent starvation by periodically rotating lanes.

3. Algorithm(s) Used

Greedy Scheduling Priority Queue
Emergency Vehicle Fastest Route Planner
Zone: City-wide ¬∑ Category: Emergency Response
View code (update link)
Graph with real-time blocked edges; heuristic uses straight-line distance to target.
A* Search

1. Problem

Ambulances and fire trucks must reach incidents as fast as possible. Some usual routes may be blocked due to traffic jams, accidents or road works.

2. Practical Solution

Using real-time road status, we run A* search (or modified Dijkstra) from the emergency station to the incident location. A* combines actual travel time so far with a heuristic estimate (straight-line distance) to prioritize promising routes and compute the quickest feasible path.

3. Algorithm(s) Used

A* Search Real-time Dijkstra Heuristic Design

Garbage Truck Route Optimization
Zone: Neighborhoods ¬∑ Category: Waste Collection
View code (update link)
Set of pickup points; truck must start and end at depot while minimizing total travel distance.
TSP Approximation

1. Problem

Garbage trucks often travel long unnecessary distances, leaving some bins full for a long time. Route planning is currently manual.

2. Practical Solution

We treat collection points as cities in a Travelling Salesman Problem. Using a greedy TSP heuristic (nearest neighbour + 2-opt improvement), we generate cyclic routes starting from the depot that keep total distance short while still visiting all pickup locations.

3. Algorithm(s) Used

Greedy TSP Heuristic 2-opt Improvement
Public Transport Timetable Generation
Zone: Bus Network ¬∑ Category: Scheduling
View code (update link)
Each bus trip is an interval with start and end time; objective is to maximize useful trips.
Interval Scheduling

1. Problem

There are limited buses but many candidate trips. Overlapping trips cause shortage on some routes and idle buses at other times.

2. Practical Solution

Treat each possible trip as an interval on a timeline. Using interval scheduling, we choose a maximum set of non-overlapping trips for every bus and generate a timetable that improves coverage with the same fleet size.

3. Algorithm(s) Used

Greedy Interval Scheduling Sorting by Finish Time

Graph of regions where some edges correspond to bridges; removing them splits the graph.
Min-Cut / Articulation

1. Problem

If certain bridges fail due to floods or maintenance, large parts of the city become disconnected. The city must know which bridges are most critical and should be strengthened first.

2. Practical Solution

We represent important road segments and bridges as a graph. Using min-cut and articulation edge ideas, we find edges whose removal disconnects key regions. These bridges are ranked and prioritized for monitoring and upgrades.

Min-Cut Bridge / Articulation Edge Detection
Flood Evacuation Simulation
Zone: Low-lying Areas ¬∑ Category: Disaster Management
View code (update link)
Levels of BFS from flooded source show time steps at which each locality becomes unsafe.
BFS Levels

1. Problem

During heavy rain, some localities get flooded quickly. Authorities need to know which areas will be affected first and how much time is available to evacuate people to safe shelters.

2. Practical Solution

We create a graph where nodes are localities and edges are road connections. Starting BFS from flood source points, each BFS level corresponds to time units until water reaches that locality. Shelters reachable before a cut-off level are marked as safe evacuation options.

3. Algorithm(s) Used

Breadth-First Search Multi-source BFS
