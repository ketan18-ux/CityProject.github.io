1. Optimal Road Network Construction
Transport Infrastructure Minimum Spanning Tree ¬∑ Kruskal + Prim + Union‚ÄìFind

Problem Context

The city is expanding with several new townships. Each pair of major junctions has an estimated road construction cost (land, materials, labour). If engineers connect junctions in an ad-hoc manner, they may create redundant loops, overspend the budget, and still leave some junctions disconnected.

We need a backbone road layout that connects all junctions with the minimum possible total cost, while guaranteeing that every junction is reachable from every other one.

Algorithmic Solution (Kruskal + Prim + Union‚ÄìFind)

The road network is modeled as a weighted, undirected graph G(V, E). Each junction is a vertex; each potential road is an edge with its construction cost as weight.

  • Kruskal‚Äôs MST: Sort all edges by increasing cost and add them one by one if they do not form a cycle. Cycle detection is done in near O(1) time using Disjoint Set Union (Union‚ÄìFind) with path compression and union by rank. Overall time: O(E log E).
  • Prim‚Äôs MST: Start from any junction and repeatedly add the cheapest edge that connects a new junction to the growing tree using a min-priority queue (heap). Time: O(E log V) with adjacency list.

In this project, we implement both methods and compare their MST cost to ensure correctness.

Input / Output (Project Version)

Input

A fixed small example with 6 junctions and 8‚Äì10 candidate roads. Each road is given as (u, v, cost). Data is reused for diagrams and C++ simulations.

Output

Road segments selected in the MST and the total construction cost. Both Kruskal and Prim should produce the same cost, which validates the solution.

Related SDGs

üå± SDG 9.1 ‚Äì Sustainable Infrastructure: Designs a cost-efficient backbone of main roads that can later be densified.
üå± SDG 11.2 ‚Äì Accessible Transport: Ensures all key junctions are connected so public transport can be deployed everywhere.

How These Algorithms Help in General

MST algorithms are used in designing electric grids, optical fiber networks, water pipelines, and campus-wide networks. Union‚ÄìFind is also widely used for connectivity checking, clustering, and detecting cycles efficiently in many graph-based problems.

2. Shortest Travel-Time Routing
Commuter Mobility Dijkstra + BFS (Time-weighted Graph)

Problem Context

During peak office hours, shortest-distance routes are not always the fastest. Narrow internal roads may be jammed, while slightly longer ring roads may be faster. Citizens need a routing engine that optimises for travel time rather than distance.

For a chosen source zone (home, workplace or station), we want the minimum travel time to every other zone and the corresponding path.

Algorithmic Solution (Dijkstra + BFS)

The city is modeled as a directed, weighted graph: nodes are zones; edges are roads with time cost (minutes) as weight.

  • Dijkstra‚Äôs Algorithm: With a min-heap (priority queue), we compute single-source shortest travel time to all zones in O((V + E) log V). This is efficient and standard for positive weights.
  • BFS for Unweighted Subgraphs: For pedestrian-only or signal-free sub-networks where all edges have equal time cost, BFS is used (O(V + E)) to quickly evaluate reachability and approximate times.

Combining both lets us handle realistic networks with a mix of simple and time-weighted segments.

Input / Output (Project Version)

Input

Adjacency list of 5‚Äì7 zones, with each road storing travel time. A source zone index is also given.

Output

For each zone: minimum travel time and optionally the reconstructed path (sequence of zones) from the source. Unreachable zones are reported separately.

Related SDGs

üå± SDG 11.2 ‚Äì Sustainable Transport: Reduces average commute time and supports efficient public transport routing.
üå± SDG 9.C ‚Äì Smart Infrastructure: Forms a core component of intelligent navigation systems and traffic management platforms.

How These Algorithms Help in General

Dijkstra is the backbone of almost every navigation system (cars, buses, trains, maps). BFS is heavily used for unweighted pathfinding and level-based exploration in communication networks, social networks, and games. Efficient shortest path algorithms are a fundamental building block for any smart-city software stack.

3. Traffic Signal Timing Optimization
Junction Management Queues + Priority Queue + Efficient Sorting

Problem Context

Fixed, equal green times at a 4-way junction result in some directions being overloaded while others remain almost empty. This increases waiting time, fuel consumption, and emissions at major intersections.

We want a smart timing plan that dynamically adjusts green durations based on queue lengths observed at each approach of the junction.

Algorithmic Solution (Queues + Sorting + Heap)

Each lane is represented by a queue of vehicles. At the start of a cycle, we know the queue length for each lane.

  • Queue lengths are collected into an array and sorted using an efficient algorithm such as Merge Sort or Quick Sort (O(k log k), where k is number of lanes).
  • A max-heap (priority queue) can be used to always pick the most congested lane quickly in O(log k) time for adaptive reallocation.
  • A base green time is given to every lane; extra green time is distributed proportionally to queue size.

This implements a greedy but efficient strategy that adapts to real-time demand while staying fast enough for live updates.

Input / Output (Project Version)

Input

Queue length for each lane (e.g., North, East, South, West) and total cycle time (e.g., 120 seconds).

Output

Suggested green time for each lane in that cycle, printed as a table (lane name vs allocated time).

Related SDGs

üå± SDG 11.2 ‚Äì Sustainable Transport: Smoother junction flow benefits buses and non-motorised transport as well.
üå± SDG 11.6 ‚Äì Reduced Environmental Impact: Lower idle times at signals reduce fuel use and urban air pollution.

How These Algorithms Help in General

Queue-based modeling is used in CPU scheduling, operating systems, and call centers. Priority queues and efficient sorting allow us to react quickly to changing loads, making them ideal tools for real-time traffic signal control and many other resource allocation problems.

4. Emergency Vehicle Fastest Route Planner
Emergency Response BFS + DFS on Reduced Graph

Problem Context

When an accident or medical emergency occurs, normal routes may be blocked by traffic jams, road work or crowding. Emergency vehicles need to quickly find which roads are still passable and select the shortest route in terms of hops or distance.

Algorithmic Solution (BFS + DFS)

We treat the road network as an unweighted graph for simplicity in this module: edges represent feasible road segments (after removing blocked roads).

  • Pre-processing with DFS: DFS is used to identify connected components and confirm that the incident location lies in the same component as the station after removing blocked edges.
  • Multi-source BFS: BFS starting from the station gives the minimum number of edges (or hops) to reach the incident. If each hop is considered approximately equal time, BFS gives the fastest route in O(V + E) time.

This combination is efficient and robust enough for small to medium-size networks and can be extended with weights later.

Input / Output (Project Version)

Input

Undirected graph with 5‚Äì7 nodes, list of edges, list of blocked roads, station node index, and incident node index.

Output

Whether a feasible route exists or not, shortest hop-count distance, and one example path (sequence of nodes) from station to incident.

Related SDGs

üå± SDG 3 ‚Äì Good Health & Well-being: Faster emergency response directly improves survival rates.
üå± SDG 11.5 ‚Äì Disaster Risk Reduction: The same framework can be reused for evacuation and fire safety planning.

How These Algorithms Help in General

BFS and DFS are core building blocks of graph theory: they are used to detect connectivity, find shortest paths in unweighted graphs, and explore states in many AI search problems. Mastering BFS/DFS is essential for any infrastructure or routing-related algorithm.

5. Water Pipeline Pressure Optimization
Water Distribution Graph Traversal + Efficient Sorting / Ranking

Problem Context

In the water distribution network, a few trunk pipelines feed many residential branches. Some tail-end colonies report very low water pressure during peak hours, while other regions receive excess supply.

We want to identify which pipelines (edges) are the most critical ‚Äúchoke points‚Äù so that they can be upgraded first within a limited budget.

Algorithmic Solution (DFS/BFS + Sorting)

Instead of implementing full max-flow (which is heavy), we adopt an efficient, syllabus-friendly approach:

  • Use BFS/DFS to compute how many consumer nodes are downstream of each pipe (roughly how many households depend on that edge).
  • Store each pipe with its usage score (downstream demand / capacity ratio) in an array.
  • Sort all pipes using an efficient sorting algorithm such as Merge Sort or Quick Sort in O(E log E) to rank them from most critical to least critical.

The top-ranked pipes are reported as the main bottlenecks where upgrades will have maximum impact on pressure.

Input / Output (Project Version)

Input

A small directed graph of pipes, capacities for each pipe, and demand values at consumer nodes.

Output

A sorted list of pipes with their criticality score, showing which ones to reinforce first to improve water pressure across the network.

Related SDGs

üå± SDG 6.1 ‚Äì Safe and Affordable Drinking Water: Prioritised pipeline upgrades help ensure reliable supply to all colonies.
üå± SDG 9.1 ‚Äì Sustainable Infrastructure: Data-driven identification of bottlenecks ensures infrastructure investments are optimally used.

How These Algorithms Help in General

Combining graph traversal (BFS/DFS) with efficient sorting is a common pattern: we first compute metrics for each component and then rank them by importance. The same approach can be applied to traffic hotspots, power line loads, and even social network influence analysis.

6. Power Grid Load Minimization
Electric Grid Prim‚Äôs MST + Heap (O(E log V))

Problem Context

A new housing layout must be connected to the main power grid. Multiple candidate cable routes exist between substations and new transformers. Each cable has a cost and expected energy loss. The goal is to connect all substations and new colonies with minimal total wiring cost and loss.

Algorithmic Solution (Prim‚Äôs MST with Priority Queue)

We model substations and colonies as nodes and cables as weighted edges (weight = cost + loss). Prim‚Äôs algorithm is ideal for this dense-ish network:

  • Start from a root substation and push all outgoing edges into a min-heap.
  • Repeatedly extract the minimum-weight edge that connects a new node to the growing tree.
  • Complexity is O(E log V), efficient for typical grid sizes.

The resulting MST represents the recommended power backbone network for the new region.

Input / Output (Project Version)

Input

An undirected graph of 6‚Äì7 substations/colonies with cable weights. Input is stored as adjacency lists.

Output

List of MST edges chosen, showing which substations/colonies get directly connected and the final total cost.

Related SDGs

üå± SDG 7 ‚Äì Affordable and Clean Energy: Efficient grid layouts reduce technical losses and cost of expansion.
üå± SDG 9 ‚Äì Industry, Innovation and Infrastructure: Supports expansion of reliable and sustainable power infrastructure.

How This Algorithm Helps in General

Prim‚Äôs MST is widely used in network design problems: computer networks, cable TV, and backbone layouts. It is particularly preferred when the graph is dense and adjacency lists are available, since it gives a near-optimal performance with heaps.

7. Garbage Truck Route Optimization
Solid Waste Dijkstra + Greedy Ordering + Sorting

Problem Context

Garbage trucks often travel long unnecessary distances because routes are manually planned or repeated daily without optimisation. This leads to high fuel cost, long collection times, and inconsistent service in some wards.

Algorithmic Solution (Shortest Paths + Greedy Selection)

We do not use full TSP here. Instead, we combine:

  • Dijkstra‚Äôs algorithm to pre-compute shortest distances between all key points (depot and pickup locations).
  • Greedy nearest-neighbour route construction: starting from depot, repeatedly go to the nearest unvisited pickup point based on precomputed distances.
  • Efficient sorting can be used to order candidate next locations or to compare alternative heuristic tours (O(n log n)).

This gives a good-quality, efficient route planning method suitable for small- to medium-sized wards.

Input / Output (Project Version)

Input

Number of pickup points, distance matrix between all locations (including depot), and starting depot index.

Output

An ordered sequence of locations forming the garbage collection tour and the total travel distance.

Related SDGs

üå± SDG 11.6 ‚Äì Waste Management: Efficient routing helps ensure bins are cleared before overflow.
üå± SDG 13 ‚Äì Climate Action: Shorter routes directly reduce fuel consumption and emissions.

How These Algorithms Help in General

Combining shortest path computation with greedy ordering is a common approach in logistics: courier deliveries, school bus routes, and supply chain optimisation. Even if the route is not globally optimal, it is much better than a human-designed ad-hoc path.

8. Public Transport Timetable Generation
Public Transport Efficient Sorting + Greedy Interval Selection

Problem Context

The city has limited buses but many possible trips. If all trips are blindly scheduled, some buses will have overlapping assignments, or buses may remain idle during busy periods.

Algorithmic Solution (Interval Scheduling with Sorting)

Each candidate bus trip is an interval [start time, end time]. We want to select a maximum number of non-overlapping trips for each bus.

  • Trips are sorted by finishing time using Merge Sort / Quick Sort in O(n log n).
  • Then we greedily pick the earliest finishing trip that does not overlap with the last chosen trip.

This is the classic optimal solution to the interval scheduling problem and is very efficient in both theory and practice.

Input / Output (Project Version)

Input

List of candidate trips with start and end times (e.g., in hours or minutes for one day).

Output

A subset of trips assigned to a particular bus (Bus 1) such that no two selected trips overlap, along with the number of trips served.

Related SDGs

üå± SDG 11.2 ‚Äì Public Transport Access: Better timetables improve reliability and service frequency for commuters.
üå± SDG 9.4 ‚Äì Resource Efficiency: Maximises utilisation of existing buses before demanding more vehicles.

How These Algorithms Help in General

Sorting + greedy interval selection is used in CPU scheduling, room booking systems, exam timetables, and many other scheduling tasks. It is a powerful simple pattern that gives mathematically optimal results with O(n log n) time complexity.

9. Bridge Construction Priority Model
Critical Infrastructure DFS-Based Bridge Detection (Tarjan Idea)

Problem Context

Some bridges and elevated roads are so critical that if they fail, large regions of the city would be disconnected from hospitals, schools, or workplaces. Authorities need to identify such ‚Äúcritical links‚Äù and prioritise them for regular inspection and strengthening.

Algorithmic Solution (DFS Bridges)

The road network is modeled as an undirected graph. We use a DFS-based bridge detection algorithm:

  • Perform DFS and maintain discovery time and lowest reachable discovery time (disc[u] and low[u]) for each vertex.
  • An edge (u, v) is a bridge if low[v] > disc[u], meaning there is no alternate path from v back to u or its ancestors.
  • The algorithm runs in O(V + E) time, which is linear and efficient.

Identified bridges are added to a priority list for monitoring and retrofitting.

Input / Output (Project Version)

Input

An undirected graph of regions connected by roads/bridges. For demo, a small network with a mix of cycles and tree-like parts is used.

Output

List of edges that are bridges (critical links). These are the edges whose removal would increase the number of connected components of the graph.

Related SDGs

üå± SDG 9 ‚Äì Resilient Infrastructure: Protects key transport links to ensure reliability.
üå± SDG 11 ‚Äì Sustainable Cities: Helps in planning safer and more robust urban road networks.

How This Algorithm Helps in General

Bridge detection is part of larger network reliability analysis. Similar ideas are applied in communication networks (finding vulnerable links), social networks (critical connections between communities), and power grids. DFS-based solutions are elegant and very efficient.

10. Flood Evacuation Simulation
Disaster Management Multi-source BFS (O(V + E))

Problem Context

During intense rainfall, low-lying areas near rivers or lakes get flooded first and the water gradually spreads to nearby localities. Authorities want to know which shelters remain reachable before floodwaters cut off roads, so that people can be moved safely.

Algorithmic Solution (Multi-source BFS)

Localities are graph nodes and roads are edges. Initial flood sources (river bank, lakefront) are inserted into a queue with distance/time = 0. Then we run multi-source BFS:

  • Each BFS level represents the time step when water reaches a new locality.
  • Arrival times at shelter nodes are compared with evacuation deadlines.

Time complexity is O(V + E), so this method easily scales to reasonably large city graphs.

Input / Output (Project Version)

Input

Undirected graph of 7‚Äì10 localities, a list of flood-source nodes, and a set of shelter nodes. A time-step threshold is given as the safe evacuation limit.

Output

Flood arrival time for each locality and classification of each shelter as ‚Äúsafe‚Äù or ‚Äúunsafe‚Äù based on whether water reaches before the threshold.

Related SDGs

üå± SDG 11.5 ‚Äì Disaster Risk Reduction: Supports planning that reduces deaths and affected people due to disasters.
üå± SDG 13 ‚Äì Climate Action: Models climate-related flood events and helps prepare adaptive responses.

How This Algorithm Helps in General

Multi-source BFS is a natural model for spreading processes: floods, fire, disease, information, or even viral rumours on social networks. Understanding wave-like propagation helps planners locate safe zones, design barriers, and schedule timely evacuations.
